               │                        │

                       CONFIGURATION & ENVIRONMENT
═══════════════════════════════════════════════════════════════════════════════

Required Environment Variables:
├── OPENAI_API_KEY           → OpenAI API authentication
├── PINECONE_API_KEY         → Pinecone vector DB authentication
├── DATABASE_URL             → SQLite path (default: sqlite:///sample.db)
├── OPENAI_MODEL             → Model to use (default: gpt-4o-mini)
├── PINECONE_INDEX           → Index name (default: final-rag-index-openai-small)
└── PINECONE_METRIC          → Metric type (default: dotproduct)

Configuration Sources (Priority Order):
1. Environment Variables (Highest Priority)
2. .env file (if project uses python-dotenv)
3. Settings class defaults (Lowest Priority)

═══════════════════════════════════════════════════════════════════════════════
                         SYSTEM CHARACTERISTICS
═══════════════════════════════════════════════════════════════════════════════

Parallelization:
  ✓ Document retrieval and context summarization run in parallel
  ✓ API requests handled asynchronously via FastAPI
  ✓ Non-blocking workflow execution using asyncio.to_thread()

Scalability Features:
  ✓ LLM request concurrency control (Semaphore, max 10 concurrent)
  ✓ Vector DB batch operations (batch size: 200)
  ✓ SQLite WAL mode for concurrent reads
  ✓ Connection pooling for database

Fault Tolerance:
  ✓ Validation of all dependencies at API startup
  ✓ Error handling in node execution
  ✓ Database transaction management (commit/rollback)

Domain Specialization:
  ✓ Financial Compliance focused prompts and system instructions
  ✓ Context-aware responses using conversation history
  ✓ Hybrid search combining dense and sparse embeddings

Extensibility:
  ✓ Pluggable LLM implementations (Protocol-based abstraction)
  ✓ Swappable embedding strategies
  ✓ Modular node design in LangGraph
